# Roles（按岗位分类的 Skills 导航）

本目录用于**用“岗位角色”来组织与导航 Skills**：你先判断“我现在处于哪个角色”，再进入对应目录查看 `NAV.md`，按推荐的技能类型去挑选/收录/使用。

> 说明：这里的“Skills”既可以是你自研的，也可以是你收藏的外部 Skills。本文档只负责**分类与导航**，不规定 Skills 的内部实现结构。

## 岗位目录

- `product_manager/`：产品经理（PM）
- `delivery_manager/`：项目经理 / 交付经理（TPM/PMO）
- `developer_engineer/`：开发工程师（前端/后端/全栈）
- `qa_engineer/`：测试工程师 / QA
- `devops_sre/`：运维 / DevOps / SRE
- `growth_marketing/`：运营 / 增长 / 市场
- `office_ops/`：日常办公（PPT/PDF/Word/Excel）
- `research_data/`：研究 / 数据 / 科研（含统计、可视化、数据库检索等）

## 使用方式（最小心智模型）

- **先选角色**：你此刻最像哪类岗位在“交付”？
- **再看产物**：该岗位最常交付的产物是什么（PRD/发布/报告/脚本/文案/PPT 等）？
- **再选技能类型**：在 `NAV.md` 里找到对应的“技能类型”，从你现有技能库里优先挑已成熟的技能家族使用。
